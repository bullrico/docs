## What I'm Documenting

I'm tracking my experiments combining Rails with AI tools. Some things work great, others... not so much. Here's what I've been exploring:

#### AI Integration Tutorials
- **Local LLM Setup**: Run Llama 2, Code Llama, and other models on your own hardware
- **FastAPI Integration**: Build Python AI services that integrate seamlessly with Rails
- **Production Architecture**: Docker containers, GPU optimization, and scaling strategies
- **Background Processing**: Using Rails 8's Solid Queue for AI jobs and long-running tasks

#### Rails + AI Architecture Patterns
- **Microservices Communication**: HTTP APIs, message queues, and error handling
- **Database Design**: Storing embeddings, model outputs, and AI metadata in PostgreSQL
- **Caching Strategies**: Using Solid Cache for expensive AI operations
- **Performance Optimization**: From Rails queries to GPU memory management

#### Real Production Examples
- **Working Code**: Every tutorial includes complete, runnable examples
- **Docker Compose**: One-command development environments
- **Error Handling**: Battle-tested patterns for dealing with AI service failures
- **Monitoring**: Logging, metrics, and debugging AI-powered features

## About Me

I'm Bobby Santiago. I've been building Rails applications since 2006 (including 6 years at SurveyMonkey). When ChatGPT launched, I realized I needed to figure out how AI fits into the web applications I've been building for years.
